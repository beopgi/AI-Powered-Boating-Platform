# AI 기반 Product-Ready 보팅 라이프 플랫폼 PRD

## 1. 제품 개요 (Product Overview)

### 1.1 제품명
- **AI 기반 Product-Ready 보팅 라이프 플랫폼**

### 1.2 제품 비전
- 사용자가 바다와 관련된 일상(여가/관광/스포츠/문화/소비)을 하나의 앱에서 끊김 없이 수행하도록 지원하는 **Coastal Lifestyle 올인원 플랫폼** 구축.
- 항해 로그와 공공데이터를 지속적으로 축적·학습하여, 사용자와 지역사회 모두에게 가치 있는 **집단지성 항해 인텔리전스(Sailing Intelligence)**를 형성.

### 1.3 서비스 정의
- 본 서비스는 모바일 앱(Flutter 기반)을 중심으로, 사용자의 항해 데이터(GPS/속도/경로), 위치정보, 해양 공공데이터(기상/해양 환경/관광 데이터), 사용자 행동 로그를 결합해 아래 핵심 경험을 제공한다.
  1) AI 맞춤 추천
  2) 항해 기록 자동 아카이브
  3) 지도 기반 탐색
  4) 유람선 이벤트 실시간 예약
  5) 커뮤니티 및 커머스/예약 결합형 Product-Ready 기능
  6) 한국어/영어 다국어 지원

### 1.4 서비스 가치 제안 (Value Proposition)
- **사용자 가치**: “오늘 바다에서 무엇을 할지”를 탐색부터 예약·기록·공유까지 한 번에 해결.
- **운영 가치**: 데이터 축적 기반 추천 정확도와 재방문율 상승.
- **생태계 가치**: 지역 해양 사업자(유람선/장비/체험 프로그램)와 사용자를 연결하는 디지털 유통 채널 제공.

## 2. 개발 배경 및 필요성 (Problem Statement & Background)

### 2.1 문제 정의
1. **항해 데이터 축적 부족**
   - 기존 서비스는 항해를 일회성 콘텐츠로 소비하며, 구조화된 항해 로그(경로/거리/속도/시간)를 장기 보존하지 않음.
2. **데이터 기반 의사결정 부재**
   - 누적 데이터가 없어 개인화 추천·안전 가이드·혼잡 회피·최적 활동 제안이 제한됨.
3. **분산된 해양 라이프 서비스**
   - 탐색, 예약, 커뮤니티, 장비/상품 구매가 개별 서비스로 분리되어 사용자 여정이 단절됨.

### 2.2 기회 영역
- 모바일 위치기반 서비스 성숙, 해양 공공데이터 개방 확대, 관광·레저의 체험 중심 소비 증가.
- “바다 중심 생활(Coastal Lifestyle)” 수요 확대에 비해 통합 서비스는 부재.

### 2.3 Coastal Lifestyle 정의
- **Coastal Lifestyle** = 바다라는 공간을 중심으로 형성되는 여가, 관광, 스포츠, 문화, 소비, 생활 습관 전반.
- 본 제품은 단순 보팅 도구가 아니라, 사용자의 주말/여행/취미/커뮤니티 소비를 통합 지원하는 라이프 플랫폼을 지향.

## 3. 제품 목표 및 KPI (Business Goals & Metrics)

### 3.1 비즈니스 목표
- **G1. 일상 사용성 확보**: 항해 당일뿐 아니라 사전 탐색/예약, 사후 기록 조회까지 매일 사용하는 앱으로 포지셔닝.
- **G2. 데이터 자산화**: 항해 로그와 활동 로그를 누적해 추천·예측·운영 최적화의 핵심 자산 구축.
- **G3. 수익화 기반 확보**: 이벤트 예약, 장비 예약, 상품 구매, 제휴 수수료 모델 정착.

### 3.2 핵심 KPI
- **활성 지표**
  - MAU, WAU, DAU
  - D1/D7/D30 리텐션
  - 주간 평균 세션 수/세션 길이
- **데이터 지표**
  - 사용자당 월 평균 항해 로그 수
  - 항해 로그 완전성(경로/속도/거리/시간 기록 성공률)
  - 위치 데이터 수집 동의율
- **추천 지표**
  - 추천 클릭률(CTR)
  - 추천 후 예약 전환율
  - 추천 만족도(별점/피드백)
- **예약/매출 지표**
  - 이벤트 예약 건수
  - 예약 성공률(실시간 좌석 반영 정확도)
  - 결제 성공률
  - GMV 및 수수료 매출
- **운영 품질 지표**
  - 앱 크래시 프리 세션 비율
  - API 에러율
  - 지도 렌더링 성공률

### 3.3 KPI 목표치 (MVP 출시 후 6개월)
- MAU 30,000 / D30 리텐션 25%+
- 월 항해 로그 100,000건 누적
- 추천 CTR 18%+
- 이벤트 예약 전환율 8%+
- 결제 성공률 97%+

## 4. 핵심 사용자 정의 (User Personas)

### Persona A: 주말 레저형 보트 입문자 (28세, 직장인)
- 니즈: 쉬운 탐색, 날씨/혼잡 고려 추천, 초보자용 체험 예약
- 페인포인트: 정보가 흩어져 있고 무엇을 선택해야 할지 모름
- 성공기준: 추천→예약→이용→기록 공유까지 10분 내 완료

### Persona B: 숙련 보팅 사용자 (39세, 동호회 운영)
- 니즈: 정밀한 항해 기록, 루트 비교, 커뮤니티 공유
- 페인포인트: 기존 앱의 기록 정확도·통계 기능 부족
- 성공기준: 항해 리포트 자동 생성, 월별/구간별 분석 제공

### Persona C: 해양 관광 여행자(외국인 포함)
- 니즈: 지역별 명소/체험/이벤트를 지도에서 즉시 확인 및 예약
- 페인포인트: 언어 장벽, 예약 채널 분산
- 성공기준: 영문 UI, 원클릭 예약, 다국어 알림

### Persona D: 파트너 사업자(유람선/체험 운영사)
- 니즈: 실시간 좌석 관리, 수요 예측, 예약 운영 자동화
- 페인포인트: 전화/메신저 기반 수기 예약으로 운영 비효율
- 성공기준: 앱 기반 판매 비중 증가, 노쇼율 감소

## 5. 사용자 여정 (Customer Journey / User Flow)

### 5.1 End-to-End 여정
1) 온보딩: 회원가입/로그인 → 언어 선택(한/영) → 위치/GPS 권한 동의
2) 홈: 오늘의 추천 카드(날씨/파도/인기 활동 반영)
3) 탐색: 지도에서 추천 장소·항해 포인트·이벤트 확인
4) 예약: 유람선 이벤트/장비 예약 → 결제 → 예약 확정 알림
5) 이용: 항해 시작 시 자동 기록 활성화
6) 종료: 항해 기록 저장/통계 생성/공유
7) 재방문: 개인화 추천 고도화 + 리마인드 알림

### 5.2 사용자 흐름 다이어그램(텍스트)
- **Flow A: 추천 기반 예약 플로우**
  - App Launch → 언어선택 → 홈 추천 노출 → 추천 상세 → 지도 확인 → 이벤트 선택 → 좌석 확인 → 결제 → 예약 완료
- **Flow B: 항해 기록 플로우**
  - 항해 시작 버튼(or 자동 감지) → GPS 트래킹 → 실시간 경로 시각화 → 항해 종료 → 서버 동기화 → 리포트 생성 → 기록 보관함 저장
- **Flow C: 탐색 중심 플로우**
  - 지도 탭 진입 → 필터 선택(관광/스포츠/문화/레저) → 포인트 선택 → 상세 정보 → 찜/공유/예약

## 6. 서비스 기능 정의 (Feature Specifications)

### 6.1 AI 기반 해양 라이프스타일 추천
- 입력 데이터: 현재 위치, 이동 패턴, 항해 로그, 기상/해양 데이터, 인기 활동, 사용 이력
- 추천 카테고리:
  - 해양 관광
  - 해양 스포츠
  - 해양 문화 활동
  - 해양 레저
  - 주변 해양 명소
- 추천 유형:
  - 상황형(“오늘 파고/풍속 기준 적합 활동”)
  - 취향형(“과거 예약/기록 기반 개인화”)
  - 트렌드형(“지역 인기 상승 활동”)
- 설명가능성: 추천 사유(예: “현재 위치 반경 5km, 유사 사용자 선호 78%”) 제공

### 6.2 항해 기록 자동 아카이브
- GPS 기반 자동 수집 항목:
  - 항해 경로(Polyline)
  - 이동 거리
  - 이동 속도(평균/최고)
  - 소요 시간
  - 출발/도착 위치
- 처리 방식:
  - 오프라인 캐시 저장 후 온라인 시 서버 동기화
  - 항해 세션 단위로 저장/조회
- 분석 기능:
  - 주간/월간 통계
  - 나의 Top 항로
  - 이동 패턴 시각화

### 6.3 지도 기반 탐색
- 지도에 표시되는 객체:
  - 항해 포인트
  - 추천 장소
  - 사용자 항해 기록
  - 해양 활동 장소
- 지도 엔진: Google Maps 또는 Mapbox API
- 핵심 UX:
  - 클러스터링, 반경 검색, 카테고리 필터
  - 상세 바텀시트(거리/예상 소요시간/예약 가능 여부)

### 6.4 유람선 이벤트 실시간 예약
- 이벤트 예시: 요가 클래스, 선상 공연, 해양 체험, 선상 파티, 교육 프로그램
- 제공 기능:
  - 이벤트 목록/상세 조회
  - 실시간 예약(좌석 재고 연동)
  - 좌석 관리(남은 좌석/구역/가격)
  - 예약 확인(바우처/QR)
  - 알림(예약 확정/변경/리마인드)
- 운영자 기능:
  - 이벤트 생성/수정/종료
  - 좌석 배정 정책 설정
  - 예약 현황 대시보드

### 6.5 Product-Ready 공통 기능
- 해양 라이프스타일 제품 구매(커머스)
- 장비 예약(보드/장비/안전장비)
- 결제 시스템(카드/간편결제)
- 사용자 계정 관리(프로필, 결제수단, 동의이력, 개인정보 설정)
- 커뮤니티(후기/사진/코스 공유)

### 6.6 다국어 지원 (한국어/영어)
- 최초 실행 시 언어 선택
- 설정 메뉴에서 실시간 언어 변경
- 전 화면 다국어 키 관리(i18n)
- 예약/알림/이메일 템플릿 다국어 제공

## 7. 기능 요구사항 (Functional Requirements)

### 7.1 요구사항 목록 (우선순위 포함)

#### P0 (MVP 필수)
- **FR-1**: 사용자는 회원가입/로그인 후 앱을 사용할 수 있어야 한다.
- **FR-2**: 앱 최초 실행 시 한국어/영어 중 언어를 선택할 수 있어야 한다.
- **FR-3**: 사용자는 설정 메뉴에서 언어를 변경할 수 있어야 한다.
- **FR-4**: 시스템은 사용자 GPS 기반 항해 경로를 자동 기록해야 한다.
- **FR-5**: 시스템은 항해 세션의 거리/속도/시간/출발/도착 정보를 저장해야 한다.
- **FR-6**: 사용자는 자신의 항해 기록 목록 및 상세 통계를 조회할 수 있어야 한다.
- **FR-7**: 시스템은 지도에서 추천 장소, 활동 장소, 항해 포인트를 표시해야 한다.
- **FR-8**: 사용자는 지도에서 카테고리 필터(관광/스포츠/문화/레저)를 적용할 수 있어야 한다.
- **FR-9**: 시스템은 위치/항해로그/공공데이터를 활용해 개인화 추천을 제공해야 한다.
- **FR-10**: 사용자는 유람선 이벤트 목록/상세를 조회할 수 있어야 한다.
- **FR-11**: 사용자는 유람선 이벤트를 실시간 좌석 기준으로 예약할 수 있어야 한다.
- **FR-12**: 시스템은 예약 확정 정보를 앱 내 및 푸시 알림으로 제공해야 한다.
- **FR-13**: 시스템은 결제를 처리하고 예약 상태를 동기화해야 한다.

#### P1 (출시 후 1차 고도화)
- **FR-14**: 사용자는 장비 예약 기능을 이용할 수 있어야 한다.
- **FR-15**: 사용자는 해양 라이프스타일 상품을 구매할 수 있어야 한다.
- **FR-16**: 시스템은 추천 사유를 사용자에게 설명해야 한다(Explainable Recommendation).
- **FR-17**: 사용자는 항해 기록을 커뮤니티에 공유할 수 있어야 한다.
- **FR-18**: 운영자는 파트너 콘솔에서 이벤트 좌석/가격/스케줄을 관리할 수 있어야 한다.

#### P2 (중장기)
- **FR-19**: 시스템은 항해 위험도 예측(기상/해양 상태 기반)을 제공해야 한다.
- **FR-20**: 시스템은 사용자 군집 기반 코스 자동 생성 기능을 제공해야 한다.
- **FR-21**: 시스템은 다국어를 추가 확장(일본어/중국어 등)할 수 있어야 한다.

### 7.2 수용 기준(샘플)
- FR-11: 동시 예약 요청 시 좌석 초과 예약이 발생하지 않아야 함(트랜잭션/락 처리).
- FR-4/5: 5초 주기 위치 수집 기준 세션 종료 후 기록 누락률 1% 미만.
- FR-2/3: 언어 변경 시 앱 재실행 없이 주요 UI 문구 즉시 전환.

## 8. 비기능 요구사항 (Non-Functional Requirements)

- **성능**
  - 지도 초기 로드 < 2.5초(4G 기준)
  - 추천 API 응답 p95 < 800ms
  - 예약 요청 처리 p95 < 1.2초
- **가용성/신뢰성**
  - 서비스 가용성 99.9%
  - 결제/예약 데이터 정합성 보장(Exactly-once 수준 처리 전략)
- **보안/개인정보**
  - OAuth2/JWT 인증
  - 개인정보 및 위치데이터 암호화(전송/저장)
  - 최소 수집·목적 제한·보관주기 정책
- **확장성**
  - 피크 트래픽(성수기/이벤트) 대응 오토스케일
- **관측성**
  - 로그/메트릭/트레이싱 통합 모니터링
  - 장애 알림 및 온콜 룰
- **컴플라이언스**
  - 국내 개인정보보호법 및 위치정보 관련 규정 준수

  - 피처 스토어

- **Integration**:
  - 지도 API(Google Maps/Mapbox)
  - 결제 게이트웨이
  - 공공 해양/기상 데이터 API

## 10. 데이터 전략 (AI 학습 및 데이터 활용)

### 10.1 데이터 소스
- 1st-party: 앱 행동 로그, 항해 세션 로그, 예약/결제 이력, 찜/리뷰
- 2nd/3rd-party: 해양 기상/파고/풍속/조류, 지역 관광 데이터

### 10.2 데이터 파이프라인
- 수집(앱 SDK/API) → 정제/익명화 → 피처 생성 → 학습/배포 → 온라인 추론 → 피드백 루프

### 10.3 AI 활용 시나리오
- 개인화 추천 랭킹
- 수요 예측 기반 이벤트 좌석/가격 운영 지원
- 위험도 안내(중장기)

### 10.4 데이터 거버넌스
- 사용자 동의 기반 수집/활용
- 민감정보 마스킹 및 접근권한 최소화
- 모델 품질/편향 모니터링

## 11. UI/UX 설계 방향

### 11.1 디자인 원칙
- **Outdoor Readability**: 야외/강한 빛에서도 가독성 높은 대비
- **One-Handed Flow**: 이동 중 조작 최소화를 위한 하단 중심 인터랙션
- **Map-First Navigation**: 지도 중심 정보 구조
- **Trust UX**: 추천 이유·예약 상태·결제 상태를 명확히 표시

### 11.2 주요 화면
- 온보딩/권한/언어 선택
- 홈(오늘의 추천)
- 지도 탐색(필터/바텀시트)
- 항해 기록(실시간 트래킹/종료 리포트)
- 이벤트 목록/상세/예약/결제
- 커뮤니티 피드
- 마이페이지/설정(언어 변경 포함)

### 11.3 접근성
- 폰트 크기 대응, 색약 대비 팔레트, VoiceOver/TalkBack 라벨링 지원

## 14. 향후 확장 전략

### 14.1 제품 확장
- 개인 맞춤형 “바다 일정 플래너”(날씨·혼잡·취향 통합)
- 웨어러블/IoT 연동(심박·안전장비 상태)
- AR 기반 마리나/포인트 안내

### 14.2 시장 확장
- 국내 주요 연안 도시 단계적 확장 → 동아시아 관광권 연계
- OTA/로컬 관광 플랫폼 제휴 통한 유입 확대

### 14.3 수익화 확장
- 예약 수수료 + 광고/프로모션 + 프리미엄 구독(고급 리포트/안전 인사이트)
- B2B 대시보드(운영사 수요예측/CRM)

### 14.4 데이터 네트워크 효과
- 사용자 증가 → 항해 데이터 정밀도 향상 → 추천 품질 개선 → 재방문 증가의 선순환 구축
